User Journeys — View & Update Flows

Developer — Docs in Repo

Writes documentation alongside code

  1. Opens .dardocs.json in VSCode
  2. Extension renders rich preview (view)
  3. Switches to edit mode in VSCode
  4. Adds Mermaid diagram, code block
  5. Commits file with code changes
  6. PR shows doc diff in GitHub
  7. Teammates view in browser or VSCode

Developer — Embedded Editor

Integrates editor into their product

  1. npm install @dardocs/editor
  2. Drops <DarDocsEditor /> into React app
  3. Configures extensions (tables, code, etc.)
  4. Users of their app create/edit documents
  5. Serializes to .dardocs.json for storage
  6. Free tier: basic text editing
  7. Paid tier: whiteboards, collab, embeds

DevOps — Incident Runbook

Maintains and executes runbooks during incidents

  1. Opens runbook in browser (SaaS or self-hosted)
  2. Reads incident procedure (view)
  3. Clicks "Run" on a bash code block
  4. Command executes against configured k8s cluster
  5. Output appears inline below the block
  6. Edits runbook based on findings (edit)
  7. Execution log saved for audit trail

PM — Living PRD

Writes specs with live data and tracks implementation

  1. Creates new PRD from template (browser)
  2. Writes requirements with rich text (edit)
  3. Embeds Mermaid architecture diagram
  4. Embeds Grafana dashboard for metrics
  5. Links GitHub PRs for each requirement
  6. Engineers view spec, see live status
  7. Stakeholders view with comment mode

Enterprise IT — Self-Hosted

Deploys and administers DarDocs on company infra

  1. Deploys via Helm chart to internal K8s
  2. Configures OIDC/SAML for SSO
  3. Connects to internal Postgres + S3
  4. Sets up role-based access controls
  5. Users access via internal URL (view/edit)
  6. Audit logs track all document operations
  7. Air-gapped — no external data flow

Startup Team — SaaS

Small team using DarDocs for all documentation

  1. Signs up at dardocs.com (free tier)
  2. Creates team workspace
  3. Writes docs, meeting notes, specs (edit)
  4. Shares links for external review (view)
  5. Upgrades for real-time collaboration
  6. Exports to .dardocs.json for backup
  7. Integrates Slack notifications
Phased Rollout
Phase 1 — Foundation

Extract & Ship the SDK

  • Monorepo: @dardocs/core + @dardocs/editor
  • npm publish with free/paid extension tiers
  • VSCode extension (view-only first)
  • Documentation site
  • License key system
Phase 2 — Platform

SaaS + Collaboration

  • Next.js app with auth (Clerk/NextAuth)
  • Hocuspocus for real-time collaboration
  • Document CRUD API + permissions
  • Stripe billing integration
  • VSCode extension: edit mode
Phase 3 — Enterprise

Self-Hosted + Verticals

  • Docker / Helm deployment packages
  • OIDC/SAML SSO, SCIM provisioning
  • Audit logging, admin console
  • Executable code blocks (runbooks)
  • API docs features, PRD templates
